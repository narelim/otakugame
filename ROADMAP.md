# 서코의 신 — 정식 게임 개발 로드맵

> 결정된 방향: **웹 기반 유지**(React + Vite). Unity 재작성 안 함 — 핵심인 그림 앱(Canvas 2D)을 살리기 위함.
> 최종 목표: **Steam(PC) + 모바일(iOS/Android)**. 온라인 교류회(2D 도트 아바타, 게더타운형) + AI API 연동.

---

## 아키텍처 (목표 형태)

```
[클라이언트]  React (UI · 그림앱 · SNS · 굿즈)          ← 현재 코드 대부분 유지
        +  PixiJS 레이어 (교류회 플로어: 2D 도트 아바타)  ← 신규
        │
        │  Electron 으로 감싸 → Steam
        │  Capacitor 로 감싸 → iOS/Android
        ▼
[백엔드]  Node + Colyseus (실시간 룸: 교류회)
        +  AI 프록시 (Claude API 호출 · 레이트리밋 · 캐싱)
        +  DB (계정 · 세이브 · 굿즈/부스 데이터)
```

기술 선정 근거:
- **PixiJS**: 게더타운도 쓰는 2D WebGL 렌더러. 도트 아바타 공유 공간에 적합.
- **Colyseus**: Node 기반 authoritative 멀티플레이 룸 서버. 교류회 방(room) 모델에 직결.
- **Electron / Capacitor**: 웹 코드 하나로 Steam + 모바일 커버. 그림 앱 재작성 0.
- **AI 프록시 필수**: API 키 노출·CORS 때문에 클라 직접 호출 불가. 멀티용 백엔드가 겸함.

### 설계 노트: 엔딩 (2026-07 결정)
- 게임 전체 엔딩은 없음(샌드박스). 엔딩은 **장르 단위의 "계정 정리"**이고 결정 주체는 플레이어.
- 정리 방식 2종: **휴덕**(계정 유지, 복귀 시 수치 일부 감쇠) / **탈덕**(계정 close + 사유 선택).
- close된 장르는 삭제가 아니라 **기록 보관소(회고록)**로: 활동 일수·총 판매·대표 굿즈·명장면 로그 영구 보존.
- 다른 장르 활동 중 옛 장르 소식이 간간히 SNS에 흘러옴 (트윗 템플릿에 archived 장르 참조 카테고리).
- **재파기(복덕)**: 탈덕한 장르를 다시 파면 "회귀 이벤트" — 새 계정으로 시작하되 옛 기록이 링크되어
  초기 팔로워 보너스(예전 팬이 알아봄) + 낮은 확률 "흑역사 발굴" 이벤트. archive 데이터는 불변, 활동 수치는 새로.
- 전체 진행 마일스톤은 "서코의 신 전당" 칭호로 (Steam 도전과제 후보).

### 콘텐츠 백로그: 바쁜 오타쿠 (2026-07)
설계 축: 유한한 시간(행동 슬롯)·골드·체력·멘탈 위에서 콘텐츠들이 **서로 경쟁** →
선택이 플레이어의 오타쿠 성향을 드러냄. 성향 통계(시간/돈 사용처)를 추적해
회고록 문장("가챠에 ₩84만 사용")과 성향 칭호("가챠의 노예"·"리셀 헌터"·"무과금 성좌")로 환원.

추가 설계 원칙 (2026-07 후반 결정):
- **활동은 특정 날짜에 발생하고 시간(행동 슬롯)을 소요한다** — 캘린더·마감과의 충돌이 곧 게임플레이.
- **장르(매체)별 주 활동이 다르다** — 소설 오타쿠는 티켓팅 대신 특화 활동(예: 연재 정주행 이벤트, 사인회 응모, 공모전)이 있어야 함. 매체별 활동 매트릭스 필요.
- 단, 장르 밖 활동도 **지인 부탁 이벤트**로 접근 가능 ("티켓팅 도와줘!" 수락 → 티켓팅 미니게임) — 인맥/메시지 시스템과 연결.

- [ ] **메루카리(중고장터) ← 백로그 1순위 (밸런싱 패스 1 결과로 승격)** — 인터넷 사이트. 구매: 절판 공식 굿즈 프리미엄 시세(덕질장 구멍 메우기 = 후반 머니싱크의 몸통)·상태등급·가품 리스크. 판매: 행사 이월 재고 판로(이월 멘탈 타격의 해소 선택지 — 밸런스 루프 완성), 가격 후려치기·사기 이벤트(메시지 연동), 리셀 논란(팬신뢰 vs 골드)
- [ ] **티켓팅** — 정해진 날짜·정각 선착순 미니게임. 성공 시 캘린더에 덕질 일정(콘서트·팝업·생카) 등록 → 행사 마감과 일정 충돌 압박의 공급원. 실패 → 메루카리 양도 티켓으로 연결
- [ ] **응모 당첨** — SNS/일상에서 원클릭 응모, 며칠 뒤 메시지로 결과 도착. 낮은 확률 × 큰 보상(멘탈 회복 + 수집품)
- [ ] **온라인 가챠** — 폰 앱(장르 IP 모바일게임 컨셉). 천장·픽뚫·10연 연출, 골드·멘탈 소모, 결과는 덕질장 수집 + mabo 자랑 포스트 연동
- [x] **성향 통계 추적 (기반)** — `logTx`에 카테고리 인자 추가, `state.stats.{spend,earn}`에 누적(세이브 v4). 현재 카테고리: goods/event/booth/daily/job — 회고록·칭호는 추후 이 데이터로
