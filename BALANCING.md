# 밸런싱 핸드오프 — 서코의 신

> 이 문서는 밸런싱 작업을 맡는 세션(Claude)을 위한 단독 브리핑입니다.
> 게임 전체 맥락은 `ROADMAP.md` 참조. 여기엔 밸런싱에 필요한 것만 담습니다.

## 0. 목표 (우선순위 순)

1. **후반 골드 인플레 해소** — Day 100+ 수익 폭주를 체감 곡선으로 (아래 관찰 ①)
2. **멘탈을 살아있는 자원으로** — 현재 200일 내내 100 고정 (관찰 ②)
3. **완판 쾌감 복구** — 현재 완판이 사실상 발생 불가 (관찰 ③)
4. 초반~중반 자금 긴장(Day 50~80 바닥권)은 **건강하니 유지** — 초반을 더 어렵게 만들지 말 것

## 1. 검증 도구 (필수 워크플로)

```bash
npm run build   # 빌드 검증 (개발 관행)
npm test        # 로직 플레이테스트 7종 — 정합성 불변식. 절대 깨지면 안 됨
# 밸런스 곡선 시뮬 (관찰용 리포트, 200일 × 페르소나 2종):
npx vitest run tests/balance.report.test.js --reporter=verbose --silent=false
```

**작업 루프**: 수치 수정 → `npm test` 통과 확인 → 밸런스 시뮬로 전/후 곡선 비교 → 커밋 메시지에 전후 요약 포함.
시뮬 리포트 읽는 법: 20일 간격 골드(k)/멘탈/체력/인지/팔로워 표 + 행사 횟수·평균 수익·완판율·최저골드·수입/지출 카테고리, 그리고 **5회 집계**(랜덤 편차 완화 — 전/후 비교는 집계 기준으로 판단할 것).
시뮬의 플레이 정책 자체(`tests/balanceSim.js`, 리포트는 `tests/balance.report.test.js`)는 "그럴듯한 플레이어"로 유지하되, 새 시스템을 추가하면 정책에도 반영할 것.

## 2.5 밸런싱 패스 1 결과 (2026-07-19)

①②③ 처방 적용 완료. 전/후는 5회 집계 평균(성실형) 기준.

- **① 골드 인플레**: fame 기여 캡 `min(0.25, fame/2000)` + fameEarned √체감 + **수요 모델** 도입 —
  `salesCap = min(maxSales, (70 + 100·fame/(fame+500)) × 규모배율(mega1.3/large1.1/medium1.0/small0.8/micro0.6) × (1+부스fame보너스) × 운(0.8~1.2))`.
  "재고를 만들수록 무한정 팔리는" 구조가 인플레의 몸통이었음. 결과: 최종골드 334만→195만, Day100+ 기울기 -46%(폭주→포화 곡선), 인지도 200일 2134→431.
- **③ 완판**: 판매율 ≥0.95 시 전량 완판. 시뮬 정책도 재고 감각(지난 판매량 기반 주문, 완판 시 증량)으로 현실화. 결과: 완판 0% → 행사당 0~2종(종 기준 10~13%). `flags.goodsSoldOut`도 이제 실제로 발화.
- **② 멘탈**: 마감 압박 D-7~D-1 -4/일(신청 행사만, D-day 배너 표기) + 이월 재고 -3/종(최대 -9) + 알바 소모(2~7) + 일상 회복 하향(원작수혈 25→15 등). 결과: 평균 99→95, 최저 80→71, 40~90 체류 5%→20% — 행사 사이클마다 골이 파임.
- **④ 초반**: 초반(≤50일) 최저골드 ₩620~1,300 — 기존 관찰(₩100~1,400) 범위 유지.

**잔여 갭 (다음 세션 몫)**:
- 최종골드가 목표(60만~120만) 대비 성실형 ~195만/올인형 ~150만. **구조적 원인**: 성실형 월급만 200일에 ~108만이라, 목표를 문자 그대로 맞추려면 서클활동 200일 순익 ≈ 0이어야 함 → 완판·성장 쾌감과 모순. 수요 모델을 더 조이는 것보다 백로그 머니싱크(메루카리·아바타 고가템·부스 확장)로 흡수하거나 월급 상한을 재논의하는 쪽을 권장.
- 멘탈 40~90 상시 진입은 미달(실제 72~100 오르내림). 압박 -4/일은 권고 밴드(-3~5)의 중간 — 더 조이는 건 유저 상의 필요 (노브: PRESSURE(eventSystem.js), 일상 행동 멘탈, 이월 재고 타격).
- 후반 아크릴 판매가 수요 소진(엽서 우선)으로 감소 — 굿즈 다변화 인센티브는 별도 과제.

## 2.6 밸런싱 패스 2 — 유저 플레이테스트 피드백 반영 (2026-07-19)

유저 피드백 "체력·멘탈이 잘 안 떨어진다" + "활동이 인게임 시간을 안 먹는다" → **활동의 시간화**로 해결.

- **알바 출근·행사 포장·커미션이 모두 행동 슬롯(ACT_MAX=2)을 소모** — 회복 행동과 경쟁하게 되어
  멘탈/체력 압박이 자연 발생. 별도 수치 너프보다 이 구조 변화가 주효.
- 신규 시스템: **포장**(D-1 미니게임 — 하면 당일 밤샘 리스크 제거+보너스, `packingSystem.js`) ·
  **커미션**(fame 10+ 시 의뢰 메시지 10%/일, 보수 5천~4.5만, 기한 3일, `commissionSystem.js` — 월급 전 부수익 루트)
- 노브: PRESSURE 4→5.
- 전/후 (성실형 5회 집계): 멘탈 최저 69→**34** · 평균 95→**80** · 40~90 체류 17%→**66%** ✦ 목표 달성.
  올인형(무직 구간)은 평균 94로 평온 — "일 안 하면 마음은 편하다"는 테마상 의도로 수용.
- 최종골드 189만→235만 (커미션 수입 추가로 상승) — **머니싱크(메루카리) 필요성이 더 커짐**. 초반 최저골드 유지.

## 2.7 백로그 콘텐츠 투입 후 (2026-07-19)

메루마켓·티켓팅·응모·가챠 투입. 시뮬 정책에도 반영 (팬이벤트 참석, 티켓팅 도전, 지름형은 가챠/프리미엄 매물/양도표, 이월 재고는 떨이).

- **머니싱크 작동 확인**: 올인형(지름형) 최종골드 170만→**49만** (가챠·프리미엄 매물·양도표로 소비) + 수집 대량 획득.

  → §5의 "머니싱크 도입 후 재평가" 조건 충족: **'벌이는 크지만 쓸 곳도 많다' 상태 달성.**

## 4. 제약 (어기면 안 됨)

- `npm test`의 정합성 검증(월급 합산, 다중 행사 승계, 불변식)은 **약화 금지**. 밸런스 수치만 바뀌면 테스트는 그대로 통과함.
- state 필드를 추가하면 `INITIAL_STATE`에 기본값 + `saveSystem.js`의 `SAVE_VERSION`/`migrate()` 규칙 준수 (기존 세이브 호환).
- UI/시스템 구조 변경은 범위 밖 — 수치·수식·데이터만. 새 머니싱크 콘텐츠(메루카리 등)는 별도 작업이므로 만들지 말 것.
- 콘텐츠 볼륨 확장(sns_events.json 대량 추가 등)도 별도 트랙 — 밸런싱과 섞지 말 것.
- 커밋: 한국어, 논리 단위 분리, 커밋 메시지에 시뮬 전/후 핵심 수치 포함. 각 커밋 전 `npm run build` + `npm test`.
- 초반(Day 1~50) 곡선을 지금보다 어렵게 만들지 말 것 (목표 4).

## 5-검증. 검증 패스 요청 노트 (2026-07-19, 패스 2·백로그 투입 직후)

이번 패스는 신규 튜닝이 아니라 **현 상태의 검증 + 필요 시 소폭 조정**입니다. 패스 1 이후 변경분은 §2.6(활동의 시간화)·§2.7(백로그 콘텐츠) 참조. 시뮬 정책(`tests/balanceSim.js`)에도 팬이벤트·티켓팅·가챠·떨이가 반영되어 있습니다.

**우선 검증해줬으면 하는 것 (의심 순):**
1. **팬 이벤트가 멘탈 압박을 무력화하는지** — 참석 보상 멘탈 +20~28은 큰 회복원. 패스 2에서 만든 마감 압박 곡선(최저 34)이 백로그 투입 후 최저 45로 완화됨. 밴드(40~90 체류) 유지 중이긴 한데, 압박이 무뎌졌다고 판단되면 노브는 FE_TYPES.mental(fanEventSystem) 또는 참석 체력 소모.
2. **가챠 기대값** — 10연 2.7만, SSR 3%, 천장 30연(≈8.1만 보장). 수집 가치 대비 과하게 싸거나 비싼지. 매물 시세(PRICE_BASE: N 1.2만~SSR 15만)와의 정합도 — 가챠가 메루마켓 구매보다 항상 우월하면 매물 탭이 죽습니다.
3. **가품 특가의 기대값** — 정상가 45% × 사기율 55%. "수학적으로 이득"이면 리스크 연출이 무의미해짐. EV가 약손해(도박 맛)인지 확인.
4. **성실형(비지름) 최종골드 247만** — 머니싱크는 '선택'이라 절약형은 여전히 쌓임. 내 판단은 "선택의 자유라 문제 없음"이지만 세컨드 오피니언 요청. 강제 소비를 만들지는 말 것.
5. **초반(Day≤50) 안전** — 티켓팅 입장료·가챠 유혹이 초반 자금(최저 ₩1,000선)을 망가뜨리지 않는지. §4 제약("초반 더 어렵게 금지") 그대로 유효.

**신규 노브 위치**: §2.7 참조 (PRICE_BASE·가챠 단가/천장·FE_TYPES·떨이 45%/중복 25%·티켓팅 성공률 표는 fanEventSystem.resolveTicketing).
**워크플로 동일**: npm test(14종) 절대 유지 → 5회 집계 전/후 비교 → 커밋 메시지에 수치.

## 5. 완료 기준 (2026-07-19 개정)

패스 1 결과를 반영해 목표를 현실화함. 원래 "최종 골드 60~120만"은 월급 구조(성실형 200일 ~108만)와
모순되어 폐기 — 서클활동 순익을 0으로 만들면 완판·성장 쾌감이 죽는다.

- **골드**: 머니싱크 콘텐츠(메루카리·아바타 프리미엄·부스 확장) 도입 전까지는 최종 150~200만 허용.
